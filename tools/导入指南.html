第一步：安装脚本依赖

打开电脑的 命令提示符（CMD）或终端，运行：

pip install python-docx

如果你的电脑有多个 Python，用完整路径：

第三步：运行转换脚本

打开终端，进入 usabo-pwa/tools/ 目录，然后运行：

🔄 追加新题目（最常用）

把新题目合并到现有题库，不影响已有题目：

python doc_to_questions.py 你的文件.docx --append

🔁 全量替换题库

用 Word 文档里的题目完整替换 questions.js 的所有题目：

python doc_to_questions.py 你的文件.docx

👁️ 仅预览不写入（安全检查）

python doc_to_questions.py 你的文件.docx --dry-run

📁 指定输出路径

python doc_to_questions.py 你的文件.docx --out C:\path\to\questions.js --append
💡 提示：脚本会自动跳过格式有问题的题目，并在终端里提示哪些题有问题，方便你回去 Word 里修改。

第四步：把更新推送到 GitHub Pages

在 usabo-pwa/ 根目录下运行：

git add questions.js git commit -m "Add new questions" git push

推送后约 1~2 分钟，https://zjun19304-stack.github.io/usabo-pwa/ 自动更新。

常见问题

❓ 题干太长怎么办？

【题干】行后面可以直接换行续写，脚本会自动合并，直到遇到选项行（A. B. ...）为止。

❓ 有图片怎么处理？

先把图片放到 usabo-pwa/images/ 目录，然后在 Word 里写 【图片】images/文件名.png 即可。

❓ 如何处理 5 个选项（A-E）？

直接写 E. 就行，脚本支持 A～F 最多 6 个选项。

❓ 脚本报错"找不到输入文件"？

确认你的 Word 文件路径正确，或者把 .docx 文件复制到 tools/ 目录再运行。
