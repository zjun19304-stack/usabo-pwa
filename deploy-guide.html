1注册 GitHub 账号

如果你已经有 GitHub 账号 → 跳到第 2 步

如果没有 → 按以下操作：

1.1 打开 GitHub 官网

在浏览器地址栏输入：https://github.com

https://github.com
GitHub
Let's build from here
Sign up

1.2 点击右上角 "Sign up"

依次填写以下信息：

字段 说明
Email 你的常用邮箱（QQ邮箱、Gmail 都行）
Password 设置密码（至少 15 个字符，或至少 8 个字符含数字和字母）
Username 起一个用户名
这个很重要！后面你的网址里就包含它
例如：usabo-teacher 或 zhang-bio
关于用户名：你的网站最终地址会是 https://你的用户名.github.io/usabo-pwa，所以选一个好记的名字。注册后可以不改，但建议一次想好。

1.3 验证邮箱

注册后 GitHub 会给你的邮箱发一封验证邮件，打开邮件点击里面的 "Verify email address" 按钮。

完成以上步骤后，你就拥有 GitHub 账号了！接下来配置本地环境。

2配置 Git 用户信息

你的电脑上已经装好了 Git，但还没告诉它"你是谁"。我们需要设置一下用户名和邮箱。

打开电脑上的 终端 / 命令行：

  • Windows：按 Win + R，输入 cmd，回车
  • Mac：打开"终端"应用

然后依次执行以下两条命令（把方括号里的内容替换成你自己的信息）：

# 设置你的用户名（用你在 GitHub 注册的用户名） git config --global user.name "你的GitHub用户名" # 设置你的邮箱（用你在 GitHub 注册的邮箱） git config --global user.email "你的邮箱@example.com"
举个例子：如果你的用户名是 usabo-teacher，邮箱是 teacher@qq.com，那就输入：
git config --global user.name "usabo-teacher"
git config --global user.email "teacher@qq.com"
设置完成后不会有什么提示，只要没报错就是成功了。

3在 GitHub 上创建新仓库

3.1 登录 GitHub，点击新建仓库

登录 GitHub 后，在页面右上角找到 "+ 号" 图标 → 点击 "New repository"

或者直接访问：https://github.com/new

3.2 填写仓库信息

设置项 填什么
Repository name usabo-pwa
（必须是这个名字，不要用中文或空格）
Description USABO Practice PWA（可选填）
Public / Private 选 Public（必须公开，GitHub Pages 免费版只支持公开仓库）
Add a README 不要勾选（我们已经有自己的文件了）
.gitignore None（不用选）
License None（不用选）

3.3 点击 "Create repository"

页面最下方绿色的 "Create repository" 按钮，点击它。

注意！创建完成后，页面会显示一段命令。你不需要复制那些命令（我下面已经准备好了），只需要记住页面上显示的仓库地址，看起来类似：
https://github.com/你的用户名/usabo-pwa.git

4把本地文件推送到 GitHub

这是最关键的一步。你的项目文件已经准备好了，现在需要把它们"发送"到 GitHub。

4.1 打开终端，进入项目目录

# 进入项目文件夹（复制粘贴即可，路径已确认） cd "C:\Users\Admin\WorkBuddy\2026-06-24-22-26-47\usabo-pwa"

4.2 提交本地文件

Git 仓库已经初始化好了，文件也已添加。现在创建第一个"提交"：

# 创建第一次提交（保存当前所有文件的快照） git commit -m "Initial commit: USABO PWA practice app"

4.3 把分支名改为 main

GitHub 默认使用 main 作为主分支名，我们把本地也改过来：

git branch -M main

4.4 关联你的 GitHub 仓库

把下面命令里的 你的用户名 替换成你 GitHub 的真实用户名，然后执行：

# 关联远程仓库（把"你的用户名"换成真实的！） git remote add origin https://github.com/你的用户名/usabo-pwa.git
例如，如果你的用户名是 usabo-teacher，就输入：
git remote add origin https://github.com/usabo-teacher/usabo-pwa.git

4.5 推送文件到 GitHub

git push -u origin main

执行后会出现以下情况之一：

情况 A：弹出登录窗口
会弹出一个 GitHub 登录页面 → 输入你的 GitHub 账号密码登录 → 授权
（如果要求输入密码，GitHub 已经不支持账号密码了，需要用 "Personal Access Token" 代替密码，见第 8 节常见问题）
情况 B：显示进度条然后成功
看到类似 Writing objects: 100% (8/8) 和 To https://github.com/... 就说明成功了！
如果报错 "failed to push" 或 "Authentication failed"，别慌！这是认证问题，看第 8 节常见问题的解决方案。

5.2 找到 Pages 设置

在左侧菜单中，找到 "Pages"（在 "Code and automation" 分类下面）

5.3 配置 Pages

  2. Branch：选择 main

"Your site is live at https://你的用户名.github.io/usabo-pwa/"

7以后怎么更新题目？

当你往题库里加了新题目（比如 Campbell 的题），只需要 3 条命令就能更新网站：

# 1. 进入项目目录 cd "C:\Users\Admin\WorkBuddy\2026-06-24-22-26-47\usabo-pwa" # 2. 提交修改（引号里写说明，随便写） git add -A git commit -m "Added Campbell questions" # 3. 推送到 GitHub（推送后 1~2 分钟网站自动更新） git push

就这三步，GitHub Pages 会自动重新部署，学生刷新页面就能看到新题目。

8常见问题

Q: push 时报错 "Authentication failed" 怎么办？

GitHub 从 2021 年开始不再支持用账号密码推送代码。你需要创建一个 Personal Access Token (PAT)：

  1. 登录 GitHub → 右上角头像 → Settings
  2. 左侧最底部 → Developer settings
  3. 左侧 → Personal access tokens → Tokens (classic)
  4. 点击 "Generate new token" → "Generate new token (classic)"
  5. Note 填写：usabo-pwa
  6. Expiration 选 No expiration（永不过期）
  7. 勾选 repo（第一个选项，勾上它下面的子项会自动全选）
  8. 页面最底部点 "Generate token"
  9. 复制生成的 token（一长串字符，类似 ghp_xxxxxxxxxxxx）
  10. 回到终端重新 push，密码处粘贴这个 token
重要：Token 只显示一次！复制后请保存到安全的地方。如果丢了，重新生成一个新的即可。

Q: 网站打开了但是空白/报错？

检查以下几点：

  • 确认仓库名是 usabo-pwa（不是 usabo-PWA 或其他大小写）
  • 确认 Pages 设置的 Branch 是 main，文件夹是 /(root)
  • 等 2 分钟再刷新（GitHub 部署需要时间）
  • 按 F12 打开浏览器控制台看有没有红色错误信息

Q: 可以用自定义域名吗？

可以！在 Pages 设置页面的 "Custom domain" 里填入你的域名，然后在域名 DNS 设置中添加 CNAME 记录指向 你的用户名.github.io。

Q: 免费够用吗？

完全够用。GitHub Pages 免费版提供：

  • 无限流量（软限制每月 100GB）
  • HTTPS 加密（自动配置）
  • 自定义域名支持
  • 仓库大小建议不超过 1GB（你的项目才几十 KB）
