密码验证 + 题库加密 + 反复制

保护效果

风险场景改造前改造后
学生查看源码直接下载 questions.js✗ 完全暴露✓ 看到的是加密乱码
学生 clone GitHub 仓库拿走题库✗ 完全暴露✓ 仓库里没有 questions.js

⚠️ 诚实说明：前端加密无法做到 100% 防破解。本方案的目标是让普通学生无法轻易获取题库，大幅提高复制门槛。对于有专业技术的用户，任何前端方案都不是绝对安全的。

工作流程

📝 整理题目 (questions.js) → 🔐 运行加密脚本 → 📦 生成 questions.enc.js → 🚀 git push → 👨‍🎓 学生输入密码使用

如何设置密码

第一步：设置密码并加密题库

打开终端，进入 usabo-pwa/tools/ 目录：

python encrypt_questions.py USABO2026

如果需要多个密码（比如不同班级用不同密码）：

python encrypt_questions.py classA,classB,teacher2026

运行后会生成 questions.enc.js（加密后的题库）。

第二步：推送到 GitHub

cd usabo-pwa git add questions.enc.js auth.js index.html app.js style.css sw.js .gitignore git commit -m "Add password protection" git push

推送后 1~2 分钟，网站自动更新。

第三步：把密码告诉学生

学生打开 https://zjun19304-stack.github.io/usabo-pwa/ 后，会看到登录界面。

输入你设置的密码（如 USABO2026）即可进入刷题。

密码保存在 sessionStorage 中——关闭浏览器后需要重新输入。

如何修改密码

只需重新运行加密脚本，用新密码：

cd usabo-pwa/tools python encrypt_questions.py MyNewPassword2026

然后推送更新：

cd .. git add questions.enc.js git commit -m "Update password" git push

所有学生需要使用新密码才能登录。

添加新题目后如何更新

当你用 doc_to_questions.py 添加了新题目到 questions.js 后，需要重新加密：

cd usabo-pwa/tools python encrypt_questions.py USABO2026

然后推送：

cd .. git add questions.enc.js git commit -m "Add new questions" git push
💡 记住：每次修改 questions.js 后，都要重新运行加密脚本生成新的 questions.enc.js。questions.js 本身不会被上传到 GitHub（已在 .gitignore 中排除）。

文件说明

文件是否上传 GitHub作用
questions.js✗ 不上传原始题库（明文），只在你电脑上
questions.enc.js✓ 上传加密后的题库，学生需要密码才能解锁
auth.js✓ 上传登录验证 + 解密逻辑
app.js✓ 上传答题逻辑（等认证通过后才启动）
tools/encrypt_questions.py✓ 上传加密工具（学生拿了也没用，没有 questions.js）

重要注意事项

1. Git 历史：如果之前已经把 questions.js 提交到 GitHub 过，即使现在 .gitignore 排除了它，历史记录里还有。需要执行：
git rm --cached questions.js 然后提交。
如果要彻底清除历史记录，需要用 git filter-branch 或 BFG Repo-Cleaner。
2. 密码安全：不要把密码写在任何会被上传的文件里。密码只在终端命令中输入，由你口头或私信告诉学生。
3. 离线使用：学生第一次输入密码后，加密题库会被 Service Worker 缓存。之后即使断网也能使用（但需要重新输入密码，因为 sessionStorage 在关闭浏览器后失效）。
✅ 总结：你的完整工作流

1. 在 questions.js 中维护题目（明文，本地）
2. 运行 python encrypt_questions.py 密码 生成加密版
3. git add . && git commit -m "update" && git push
4. 学生用密码登录使用 🎉
